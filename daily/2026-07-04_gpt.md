## Knowledge

### CMake 构建流程

```text
源码
   │
   ▼
cmake ..
   │
   ▼
生成 Makefile
   │
   ▼
make
   │
   ▼
生成可执行文件
   │
   ▼
ctest / make test
   │
   ▼
运行测试
```

核心认识：

* `cmake`：生成构建系统（不是编译）。
* `make`：真正编译代码。
* `make test`：运行测试程序（CTest）。

---

### Docker 多阶段构建

```dockerfile
FROM xxx AS base
```

作用：

* 指定基础镜像。
* 为当前构建阶段命名。
* 后续可以通过 `COPY --from=base` 或 `FROM base` 复用该阶段。

学习了：

* `add_option`

* `required()`
* `default_val()`

### 工程化开发

不要把所有逻辑写在 `main.cpp`。

推荐结构：

```text
project/
│
├── include/
├── src/
├── tests/
├── examples/
├── CMakeLists.txt
└── README.md
```

业务逻辑、参数解析、配置、网络通信分别放到不同模块。

```text

   ↓
```

* 为什么这样组织目录？

* 为什么使用 CMake？
* 为什么编写测试？
* 为什么采用这种设计？

## Framework

### 阅读一个 C++ 开源项目的方法

```text
1. Clone 项目
        │
2. 阅读 README
        │
3. 编译项目
        │
4. 运行测试
        │
5. 阅读 examples
        │
6. 阅读 CMakeLists
        │
7. 阅读 include（公开接口）
        │
8. 阅读 src（实现）
        │
9. 修改一个功能
        │
10. 提交自己的 Branch
```
