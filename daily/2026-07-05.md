## Knowledge

### AI Native

AI Native 并不是学习某个 AI 框架，而是一种全新的软件开发方式。

传统开发：

```
需求
↓
写代码
↓
测试
↓
上线
```

AI Native：

```
需求
↓
Specification
↓
AI 实现
↓
Human Review
↓
Evaluation
↓
Deploy
```

工程师逐渐从「代码实现者」转变为「AI 工作流设计者」。

---

### MCP（Model Context Protocol）

今天最大的技术收获。

MCP 更像 AI 世界的 **USB-C**。

它统一了 AI 与工具之间的通信协议，使不同 AI Client（Claude、Codex、Cursor 等）都可以调用同一个 MCP Server，而无需为每个平台重复开发。

理解了 MCP Server 的本质：

- 暴露 Tool
- 描述 Tool Schema
- 接收调用
- 返回结果

它不是框架，而是一种协议。

### 企业真正需要的能力

企业越来越关注：

- 软件工程基础
- Tool Calling
- Agent Workflow
- Evaluation
- MCP
- AI Native Workflow

相比之下，仅会调用 OpenAI API 已经成为基础能力，而不是竞争优势。

## Insight

今天最大的认知升级：

> **未来 AI 工程师最重要的能力，不是写代码，而是组织 AI 写代码。**

真正的 AI Native 工程师需要：

- 能拆解复杂需求
- 能设计完整 Workflow
- 能编写高质量 Specification
- 能验证 AI 输出
- 能保证系统稳定可靠

Coding 不会消失，但会逐渐成为整个开发流程中的一个环节，而不是全部。
